# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| SDK 1.x | Yes |
| SDK 0.x (pre-release) | No |

---

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Report security issues to: **security@zhilicon.ai**

Include:
- Description of the vulnerability and its impact
- Steps to reproduce
- SDK version and hardware/software environment
- Any proof-of-concept code (if safe to share)

---

## Response Timeline

| Stage | Target |
|-------|--------|
| Acknowledgement | Within 2 business days |
| Initial assessment | Within 5 business days |
| Fix or mitigation | Depends on severity |
| Public disclosure | Coordinated with reporter |

---

## Scope

This policy covers:
- The Zhilicon SDK (Python and C++ packages)
- Code in this examples repository
- The developer portal at developers.zhilicon.ai

Out of scope: third-party dependencies, model weights, and hardware security issues (report those separately to security@zhilicon.ai with subject "HARDWARE:").

---

## Safe Harbor

Zhilicon will not pursue legal action against researchers who report vulnerabilities in good faith following this policy.
